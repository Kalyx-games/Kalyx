## Architecture BGG (décidée, risque n°1 traité)

- **Depuis le 2 juillet 2025, l'API XML de BGG exige un enregistrement + jeton Bearer** (`Authorization: Bearer <token>`). Sans jeton → 401. Constaté en direct le 2026-07-08. Enregistrement : compte BGG puis formulaire `https://boardgamegeek.com/applications/create` (l'utilisateur devra le faire — étape manuelle).
- L'API ne renvoie **aucun en-tête CORS** → inaccessible en fetch direct depuis le navigateur, et le jeton ne doit pas être exposé côté client.
- **Décision : Edge Function Supabase comme proxy** — garde le jeton en secret, gère le retry sur 202 (« processing »), limite le rythme (throttling BGG réel, 429 possibles), et renvoie du JSON propre au client (parser le XML côté Edge Function).
- Endpoints : recherche `xmlapi2/search?query=NOM&type=boardgame` ; fiche `xmlapi2/thing?id=ID&stats=1`.
- Champs à extraire de `thing` : name (primary), minplayers, maxplayers, playingtime/minplaytime/maxplaytime, image, `statistics→ratings→averageweight` (complexité 1–5), et `players_best` via le sondage `suggested_numplayers` (valeur "Best" la plus votée ; peut être vide → null). **Vérifier le format XML réel avant de parser.**
- Les URLs d'images BGG (cf.geekdo-static.com) restent affichables directement (hotlink CDN, pas d'auth).

## Modèle de données (table `games` sur Supabase)

id uuid pk · bgg_id int null · name text · players_min int · players_max int · players_best int null · duration_min int null · duration_max int null · complexity numeric null (1–5) · image_url text null · owner text (libre, mais l'UI suggère les owners existants) · status text ('collection'|'wishlist') · created_at timestamp. **Pas d'auth, RLS désactivé ou policies totalement ouvertes.** L'utilisateur colle l'URL + anon key dans `.env` (documenter dans README).

## Spec des vues (3)

1. **Collection** (principale) : cartes compactes (image, nom, joueurs, durée, complexité, propriétaire) ; recherche live par nom ; filtres repliables (propriétaire, nb joueurs avec case « optimal » qui filtre sur players_best, durée, complexité) + bouton réinitialiser ; tris nom/aléatoire/complexité/durée ; boutons bascule wishlist↔collection, édition, suppression.
2. **Wishlist** : identique sur status='wishlist'.
3. **Statistiques** : répartitions par propriétaire/joueurs/durée/complexité (recharts), total, moyennes.

Workflow d'ajout : saisie nom → recherche BGG → liste résultats (nom + année) → sélection → fiche pré-remplie → **tous les champs éditables** avant/après enregistrement.

## Hors ligne (voulu simple)
